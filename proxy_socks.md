Проверить, что клиент работает, можно несколькими способами:

###  **Способ 1: Использовать curl с явным указанием прокси (самый надёжный)**
В вашем логе видно, что клиент поднял:
* **HTTP-прокси** на `127.0.0.1:8090`
* **SOCKS5-прокси** на `127.0.0.1:1090`

Используйте одну из этих команд:

```bash
# Через HTTP-прокси (порт 8090)
curl --proxy http://127.0.0.1:8090 ifconfig.co

# Через SOCKS5-прокси (порт 1090)
curl --socks5 127.0.0.1:1090 ifconfig.co
```

**Результат должен показать IP-адрес вашего сервера (150.20.10.56).**

### **Способ 2: Настроить переменные окружения для терминала**
Чтобы все команды в текущей сессии терминала использовали прокси:

```bash
# Для HTTP/HTTPS трафика
export HTTP_PROXY=http://127.0.0.1:8090
export HTTPS_PROXY=http://127.0.0.1:8090

# Или для SOCKS5
export ALL_PROXY=socks5://127.0.0.1:1090
```

После этого запустите:
```bash
curl ifconfig.co
```
**Снова проверьте IP.**

Чтобы отключить прокси для текущей сессии:
```bash
unset HTTP_PROXY HTTPS_PROXY ALL_PROXY
```

### **Способ 3: Проверка через браузер**
Настройте браузер на использование прокси:
1. **Настройки сети** → **Ручная настройка прокси**
2. **HTTP-прокси**: `127.0.0.1`, порт: `8090`
3. **SOCKS5**: `127.0.0.1`, порт: `1090`

После этого зайдите на сайт `ifconfig.co` или `ipleak.net`.

### **Способ 4: Проверка скорости (особенность Hysteria)**
Hysteria славится устойчивостью к потерям пакетов. Проверьте скорость через прокси:
```bash
# Установите speedtest-cli если нет
pip install speedtest-cli

# Запустите через HTTP-прокси
export HTTP_PROXY=http://127.0.0.1:8090
speedtest-cli
```

### 📝 **Что делать, если IP не меняется:**
1. **Убедитесь, что команда curl содержит флаг `--proxy`**.
2. **Проверьте конфигурацию клиента `client.yaml`**:
```yaml
# Должны быть разделы:
http:
  listen: 127.0.0.1:8090  # или другой порт

socks5:
  listen: 127.0.0.1:1090  # или другой порт
```
3. **Проверьте, не блокирует ли брандмауэр** локальные подключения к портам 8090/1090.
4. **Используйте `netstat` или `ss`**:
```bash
# Должны увидеть прослушивание портов
netstat -tulpn | grep -E '8090|1090'
# или
ss -tulpn | grep -E '8090|1090'
```

### 🧪 **Дополнительные тесты:**
- Проверьте доступность через разные сервисы:
```bash
curl --proxy http://127.0.0.1:8090 https://api.ipify.org
curl --socks5 127.0.0.1:1090 https://ipinfo.io/ip
```
- Проверьте, что DNS-запросы тоже идут через прокси (важно для предотвращения утечек):
```bash
curl --proxy http://127.0.0.1:8090 https://www.google.com
```

### 💡 **Для удобства: создайте алиас в `~/.bashrc` или `~/.zshrc`:**
```bash
alias hycurl='curl --proxy http://127.0.0.1:8090'
```
Теперь можно просто: `hycurl ifconfig.co`

Если после всех проверок IP остаётся вашим, а не серверным, возможно, есть проблема с маршрутизацией трафика в клиенте. В этом случае проверьте логи клиента на наличие ошибок при выполнении запросов.
